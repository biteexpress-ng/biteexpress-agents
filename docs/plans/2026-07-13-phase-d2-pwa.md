# Agent Program Phase D2 (PWA) Implementation Plan — Challenge UI

> **For agentic workers:** Execute task-by-task in order; steps use checkbox (`- [ ]`) syntax. Read `PRODUCT.md` and `DESIGN.md` first; load `frontend-design`, `impeccable`, and `ui-ux-pro-max` if available. The established quality floor binds verbatim.

**Goal:** Agents see their weekly challenge: live progress toward each tier, the cash on the line, time left in the week, and their past wins — plus the home-screen strip that keeps the challenge in daily view.

**Architecture:** Extends the existing app. One new route `/challenges` linked from a home progress strip (bottom nav stays at five — the strip is the entry point). Everything driven by `GET /challenge`; when `active` is false the feature is invisible everywhere.

**Tech Stack:** unchanged.

## Global Constraints

- **Repo:** `C:\laragon\www\biteexpress-agents`.
- This is the one surface where momentum/encouragement styling is on-brand (PRODUCT.md: "earned pride and momentum") — but bonuses are still money: amounts stay tabular ink-900, and the challenge card never overpromises ("Get" only when achieved; "on track" language otherwise). No countdown-pressure dark patterns: the week-ends timer is informational (day + time), not a red ticking clock.
- Bonus payout copy must match reality: awards are credited Monday morning to the AVAILABLE balance (not pending) — "Paid to your balance on Monday".
- Verification: `npm run build` per task; final seeded live pass mandatory. Dev servers via `taskkill /F /T /PID` only.
- Commit per task; push at the end. Stop after Task 4.

## API contract (frozen by D1)

`GET /challenge` (auth) → `{active, current: {week_key, week_ends_at, signups, activations, tiers: [{name, signup_target, activation_target, bonus_amount, achieved}], achieved_tier} | null, past_awards: [{week_key, tier_name, bonus_amount, signups, activations, awarded_at}]}`. Tiers ascending by signup_target; `week_ends_at`/`awarded_at` ISO-8601 UTC; `past_awards` newest-first ≤12; `active` false → `current` null, `past_awards` [] → hide the feature entirely.
