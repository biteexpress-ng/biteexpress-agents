# Agent Program Phase B2 (PWA) Implementation Plan — Customers, Earnings, Assisted Signup

> **For agentic workers:** Execute task-by-task in order; steps use checkbox (`- [ ]`) syntax. Before any UI work read `PRODUCT.md` and `DESIGN.md` at this repo's root; load the `frontend-design`, `impeccable`, and `ui-ux-pro-max` skills if available. The quality floor in the A2 plan's Global Constraints carries over verbatim (contrast ≥4.5:1, 44px+ targets, skeletons not spinners, reduced-motion alternates, tabular-nums on money/counters, lucide icons only, errors surfaced verbatim from the API).

**Goal:** Certified agents can see their referred customers, watch their earnings ledger and balances, and sign up a customer on the spot via OTP — completing the "agents earn" loop in the PWA.

**Architecture:** Extends the A2 Next.js app in this repo. Three new authed routes (`/customers`, `/earnings`, `/signup-customer`), bottom nav grows to five items, home screen swaps its placeholder for real numbers. All data from the frozen B1 API.

**Tech Stack:** unchanged from A2 (Next.js 16, React 19, TS 5, Tailwind v4, zustand, RHF+zod, lucide-react).

## Global Constraints

- **Repo:** all code in `C:\laragon\www\biteexpress-agents`.
- **Design contract:** PRODUCT.md principle 3 governs this whole phase — *numbers are the interface*. Money renders in ink-900 (never brand red; red = action), credited events may use `--color-success-strong`, amounts in tabular-nums with the ₦ prefix, two decimals.
- Verification: `npm run build` per task; the final task is a **seeded-session live pass** (standing rule from the A2 hydration bug: a green build is not a green run). Dev servers stopped only via `taskkill /F /T /PID`.
- Commit per task. Stop at the end of Task 7 — no Phase C/D features.

## API contract (frozen by B1 — do not renegotiate)

All under `/api/v1/agent`, Bearer auth:

- `GET /customers?page=N` → `{customers: [{id, name_masked, signup_channel: "code"|"assisted", status: "registered"|"activated", joined_at, first_order_at|null, orders_count, commission_total}], stats: {total, activated}, pagination: {current_page, last_page, per_page, total}}`
- `GET /earnings?page=N` → `{balances: {withdrawable, pending, earned_total}, ledger: [{id, type: "order_commission"|"reversal"|"challenge_bonus"|"manual_bonus", status: "confirmed"|"reversed", amount, order_id|null, note|null, created_at}], pagination: {...}}` — newest first; reversals are negative-amount rows; balances already net.
- `POST /customers/assisted/initiate {phone}` → 200; 409 `{code:'phone-exists'}`; 403 `{code:'not-certified'|'program-inactive'}`
- `POST /customers/assisted/confirm {phone, otp, first_name, last_name}` → 200 with masked name; 400 bad/expired OTP
- `GET /quiz/info` → `{question_count, pass_mark, cooldown_minutes}` — **pass_mark and score are percentages (0–100), always**
- `POST /quiz/submit` failure now includes `retry_at` (ISO8601)
- `GET /code/{code}/validate` (public) → `{valid, agent_name}` — not consumed by this app; listed for completeness.
