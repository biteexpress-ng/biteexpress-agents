# Agent Program Phase A2 (PWA Shell) Implementation Plan

> **For agentic workers:** Execute task-by-task in order; steps use checkbox (`- [ ]`) syntax. Before writing ANY UI code you MUST read `PRODUCT.md` and `DESIGN.md` at this repo's root — they are the design contract — and, if available in your session, load the `frontend-design`, `impeccable`, and `ui-ux-pro-max` skills. If those skills are unavailable, the distilled rules in PRODUCT.md/DESIGN.md still bind you.

**Goal:** The agents.bite.express PWA covering Phase A: login, password setup, training videos, certification quiz, and a home screen with the referral code — visually consistent with app.bite.express.

**Architecture:** Next.js App Router PWA in this repo (`biteexpress-agents`), mirroring `biteexpress-web-app` conventions (Tailwind v4 `@theme` tokens, zustand, react-hook-form + zod, lucide-react). Consumes only the Phase A1 API at `/api/v1/agent/*`. Mobile-first single column; bottom nav appears only after certification.

**Tech Stack:** Next.js 16, React 19, TypeScript 5, Tailwind CSS 4, zustand 5, react-hook-form + zod, lucide-react. Same versions as `biteexpress-web-app/package.json`.

## Global Constraints

- **Design contract:** `PRODUCT.md` + `DESIGN.md` in this repo. Tokens are PORTED from `C:\laragon\www\biteexpress-web-app\src\app\globals.css` — copy the `@theme` block, do not invent colors. Fonts: DM Sans (everything) + DM Serif Display (two display moments only), loaded via `next/font` exactly as `biteexpress-web-app/src/lib/fonts.ts` does.
- **API base URL:** `NEXT_PUBLIC_API_BASE_URL` env var; production `https://dashboard.bite.express`, local `http://dashboard.bite.express.test`. All endpoints under `/api/v1/agent/`. Auth = `Authorization: Bearer <token>` header. Error shape: `{errors: [{code, message}]}`; quiz-flow errors: `{code, retry_at?}` with HTTP 403.
- **Verification style (standing environment rule):** `npm run build` after each task; NEVER probe by force-killing dev servers. If a dev server must be stopped use `taskkill /F /T /PID <pid>`.
- **Quality floor (from the loaded design skills):** body text contrast ≥4.5:1; touch targets ≥44px (buttons/inputs 48px); visible focus rings; `prefers-reduced-motion` alternatives for every animation; base font 16px; semantic input types (`email`, `tel`); labels above inputs, never placeholder-only; skeletons not spinners; no emoji as icons (lucide only); no side-stripe card accents; no gradient text; `min-h-dvh` not `100vh`; tabular-nums on all numbers.
- Copy register: sentence case, plain verbs, encouraging not patronizing (see PRODUCT.md). Errors state cause + fix.
- Commit after every task with a conventional message. Do not deploy; Vercel hookup is a separate ops step at the end.

## API contract (frozen by Phase A1 — do not renegotiate)

| Endpoint | Auth | Request | Response (200) |
|---|---|---|---|
| `POST /auth/setup-password` | no | `{email, token, password, password_confirmation}` | `{message}`; 400 `{errors}` invalid/expired |
| `POST /auth/login` | no | `{login, password, platform?}` | `{token, agent}`; 401/403 `{errors}` |
| `POST /auth/logout` | yes | — | `{message}` |
| `GET /me` | yes | — | `{agent}` |
| `GET /training/videos` | yes | — | `{videos: [{id, youtube_video_id, title, sort_order, duration_seconds, watched}], training_complete}` |
| `POST /training/videos/{id}/watch` | yes | — | `{watched, training_complete}` |
| `GET /quiz/start` | yes | — | `{attempt_id, pass_mark, questions: [{id, question, options[4]}]}`; 403 `{code: training-incomplete\|cooldown\|no-questions\|already-certified, retry_at?}` |
| `POST /quiz/submit` | yes | `{attempt_id, answers: {qid: index}}` | `{status: passed\|failed, score, pass_mark, certified, referral_code}` |

`agent` object: `{id, full_name, email, phone, status, referral_code, training_complete, certified}`.

**Steps:**

- [ ] `src/lib/api/types.ts` — mirror the frozen contract:

```ts
export interface AgentProfile {
  id: number;
  full_name: string;
  email: string;
  phone: string;
  status: "pending" | "approved" | "certified" | "suspended";
  referral_code: string | null;
  training_complete: boolean;
  certified: boolean;
}

export interface ApiError {
  code: string;
  message: string;
}

export class ApiRequestError extends Error {
  constructor(
    public status: number,
    public errors: ApiError[],
    public raw?: unknown,
  ) {
    super(errors[0]?.message ?? `Request failed (${status})`);
  }
}

export interface TrainingVideo {
  id: number;
  youtube_video_id: string;
  title: string;
  sort_order: number;
  duration_seconds: number | null;
  watched: boolean;
}

export interface QuizQuestion {
  id: number;
  question: string;
  options: string[];
}

export interface QuizStart {
  attempt_id: number;
  pass_mark: number;
  questions: QuizQuestion[];
}

export interface QuizResult {
  status: "passed" | "failed";
  score: number;
  pass_mark: number;
  certified: boolean;
  referral_code: string | null;
}

/** 403 body from the quiz flow */
export interface QuizGateError {
  code: "training-incomplete" | "cooldown" | "no-questions" | "already-certified";
  retry_at?: string;
}
```

- [ ] `src/lib/api/client.ts` — fetch wrapper: prefixes `NEXT_PUBLIC_API_BASE_URL` + `/api/v1/agent`, sets `Accept: application/json`, attaches Bearer token from the auth store when present, parses JSON, throws `ApiRequestError` on non-2xx **including the message body** (lesson from the web-app checkout bug: never swallow error bodies — surface `errors[0].message` to the UI). On a 401 from any authed call, clear the store and redirect to `/login`.
- [ ] `src/lib/api/agent.ts` — the eight typed endpoint functions over the wrapper.
- [ ] `src/stores/auth.ts` — zustand + `persist` middleware; `hydrated` flag set via `onRehydrateStorage` so guards can wait for storage before redirecting.
- [ ] Verify: `npm run build`. Commit: `feat: typed API client and auth store`.
