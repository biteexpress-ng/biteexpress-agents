## API contract (frozen by C1 — do not renegotiate)

All under `/api/v1/agent`, Bearer auth:

- `GET /kyc` → `{kyc_status: "incomplete"|"pending"|"verified"|"rejected", rejection_reason|null, photo_url|null, identity_type|null, identity_image_url|null, bank: {bank_name|null, account_number_masked|null, account_name|null}}`
- `POST /kyc` (multipart: `photo`, `identity_type` ∈ nin|drivers_license|voters_card|passport, `identity_image`, `bank_name`, `bank_account_number` [exactly 10 digits], `bank_account_name`) → 200 `{message, kyc_status:"pending"}`; 409 `{code:"kyc-locked"}`; 422 `{errors}`. Allowed only from `incomplete`/`rejected`.
- `POST /withdraw {amount}` → 200 `{request:{id, amount, status:"pending", created_at}}`; 403 `{code: "kyc-not-verified"|"below-minimum"|"insufficient-balance"|"request-pending"}`; 422 on bad amount.
- `GET /withdrawals?page=N` → `{withdrawals:[{id, amount, status:"pending"|"approved"|"denied", admin_note|null, created_at, processed_at|null}], pagination}` — newest first, separate from the commission ledger.
- `GET /earnings` now includes `eligibility: {can_withdraw, min_amount, kyc_status, reason: null|"kyc-not-verified"|"below-minimum"|"request-pending"}` and a meaningful `balances.pending`.

### Task 1: API layer + shared pieces

**Files:**
- Modify: `src/lib/api/types.ts`, `src/lib/api/agent.ts`

**Steps:**

- [ ] Types + functions for the four endpoints (`KycStatusResponse`, `submitKyc(FormData)`, `requestWithdraw`, `getWithdrawals`, extended `EarningsResponse` with `eligibility`). `submitKyc` posts `FormData` (no JSON content-type header — let the browser set the multipart boundary); surface 422 field errors mapped to form fields and 409/403 `{code}` as typed gate errors.

Reused by earnings CTA, withdraw form, and profile.
- [ ] Verify: `npm run build`. Commit: `feat: C1 API types and gate-code copy map`.

### Task 2: KYC flow on profile

**Files:**
- Modify: `src/app/(app)/profile/page.tsx`
- Create: `src/components/kyc/kyc-card.tsx`, `src/components/kyc/kyc-form.tsx`, `src/components/kyc/image-field.tsx`

**Steps:**

- [ ] `KycCard` is status-driven, replacing the A2 placeholder note:
  - `incomplete`: "Set up payouts" intro (why: "so we know where to send your money") + the form.
  - `pending`: calm waiting state — what was submitted (photo thumbnail, ID type, masked bank), "We're reviewing your details — usually within a business day." No resubmit.
  - `rejected`: the `rejection_reason` verbatim in an alert, then the form pre-filled with bank fields for resubmission.
  - `verified`: quiet success summary (verified badge, masked account, bank name) + "To change your bank details, contact support." (v1 manual path).
- [ ] `ImageField`: label + native file input (`accept="image/*"`, `capture` hint: `user` for the passport photo, `environment` for the ID document), client-side preview, 2MB client check with a plain error, replace/remove actions. 48px+ tap targets.
- [ ] `KycForm` (RHF+zod): passport photo, ID type (radio cards, four options with plain names — "National ID (NIN)", "Driver's licence", "Voter's card", "International passport"), ID image, bank name (text input v1 — no bank-list dependency), 10-digit account number (`inputMode="numeric"`, digits-only zod), account name ("exactly as your bank knows you"). Submit → loading → success flips the card to `pending`. 422 errors land under their fields; 409 shows the `kyc-locked` copy.
- [ ] Verify: `npm run build`. Commit: `feat: status-driven KYC flow on profile`.
